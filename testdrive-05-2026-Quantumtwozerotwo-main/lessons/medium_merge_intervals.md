# Среднее задание: Merge Intervals

Нужно реализовать функцию:

```python
def merge_intervals(intervals: list[tuple[int, int]]) -> list[tuple[int, int]]:
```

На вход приходит список отрезков `[start, end]`, записанных как кортежи `(start, end)`.
Нужно объединить все пересекающиеся отрезки.

## Примеры

```python
merge_intervals([]) == []
merge_intervals([(1, 3)]) == [(1, 3)]
merge_intervals([(1, 3), (2, 5)]) == [(1, 5)]
merge_intervals([(1, 2), (5, 7)]) == [(1, 2), (5, 7)]
merge_intervals([(5, 7), (1, 3), (2, 4)]) == [(1, 4), (5, 7)]
```

Идите по шагам:

1. Пустой список возвращает пустой список.
2. Один отрезок возвращается как есть.
3. Два непересекающихся отрезка возвращаются без изменений.
4. Два пересекающихся отрезка сливаются в один.
5. Добавьте случай, где вход не отсортирован.
6. Добавьте три отрезка, из которых сливаются только два.
7. Добавьте цепочку пересечений, например `(1, 2), (2, 4), (4, 6)`.
8. Если останется время, договоритесь, считаются ли касающиеся отрезки пересечением.

Порядок для пары:

1. Откройте `medium_merge_intervals_task/`.
2. Запустите `pytest -q` и получите первый красный тест.
3. `driver` пишет минимальный код для пустого списка.
4. Когда тест зелёный, роли меняются.
5. `navigator` выбирает следующий пункт из списка выше.
6. Новый `driver` пишет следующий тест.

На этой задаче легко начать писать алгоритм целиком.

После каждого теста задавайте вопрос:
"Какой тест-кейс мы сейчас проверяем?"

Обычно удобно так:

1. отсортировать отрезки по началу;
2. пройти слева направо;
3. либо расширять последний отрезок в ответе,
4. либо добавлять новый.
