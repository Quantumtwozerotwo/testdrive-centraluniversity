# Сложное задание: DFS на сетке

Нужно реализовать функцию:

```python
def has_path(grid: list[str]) -> bool:
```

`grid` — это прямоугольная сетка из символов:
- `S` — старт;
- `G` — цель;
- `.` — свободная клетка;
- `#` — стена.

Можно ходить вверх, вниз, влево и вправо.

Нужно вернуть `True`, если из `S` можно дойти до `G`, и `False`, если нельзя.

## Пример

```python
grid = [
    "S..",
    ".#.",
    "..G",
]

has_path(grid) == True
```

Идите по шагам:

1. `S` стоит рядом с `G`.
2. Есть прямой путь по одной строке.
3. Есть путь с поворотом.
4. Стена блокирует проход.
5. Пути нет.
6. Нельзя ходить по диагонали.
7. Нельзя выходить за границы поля.
8. На поле есть цикл, и программа не должна ходить по нему бесконечно.

Порядок для пары:

1. Откройте `hard_grid_dfs_task/`.
2. Запустите `pytest -q` и получите первый красный тест.
3. `driver` пишет минимальный код для случая `["SG"]`.
4. Когда тест зелёный, роли меняются.
5. `navigator` выбирает следующий пункт из списка выше.
6. Новый `driver` пишет следующий тест.

Здесь важно:
- `driver` пишет маленький тест и короткий код;
- `navigator` следит, чтобы вы не начали писать всё решение сразу.

Пусть `navigator` говорит на уровне идеи:
- "нам нужно отмечать посещённые клетки";
- "давай сначала проверим самый простой путь";
- "теперь давай докажем тестом, что мы не зацикливаемся".

Можно выделить маленькие функции:

```python
def find_start(grid): ...
def neighbors(grid, row, col): ...
```

Если идёте слишком быстро, можно усложнить:
- вернуть не только `True` или `False`, но и сам путь;
- посчитать количество посещённых клеток;
- сделать вариант через рекурсию и вариант через свой стек.
